# security policy

## supported versions
Ce projet est en développement actif. Aucune version n’est officiellement “supportée” pour le moment.

## signaler une vulnérabilité
Merci de **ne pas** ouvrir d’issue publique pour une vulnérabilité.

- Décris le problème, l’impact, et un scénario de reproduction minimal.
- Indique la version/commit et ton environnement (OS, docker, versions).

**Canal de contact :**
- ouvre une issue **privée** si ton hébergement GitHub le permet,
- sinon contacte le mainteneur par un canal hors-public (courriel / message direct).

## divulgation responsable
- accusé de réception dès que possible
- correctif prioritaire si l’impact est sérieux
- publication d’un avis après correctif (si pertinent)
